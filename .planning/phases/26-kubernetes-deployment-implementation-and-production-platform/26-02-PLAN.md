<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API pods -> background side effects | API replicas must not duplicate scheduled publish/listener/retry work. |
| Worker env flags -> runtime behavior | Incorrect flag defaults can either stop background processing or duplicate side effects. |
| Kubernetes scaling -> business integrity | Scaling side-effecting workers without coordination can duplicate messages, chain events, or notification retries. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-04 | Tampering | worker env flags | mitigate | Use explicit env flag names and grep-verifiable disabled log messages in each processor. |
| T-26-05 | Repudiation | background processing ownership | mitigate | Document worker responsibilities and API-vs-worker values in `docs/workers.md`. |
| T-26-06 | Denial of Service | duplicate outbox/blockchain/payment/notification workers | mitigate | Default worker replicas to `1` and require locking, leader election, or idempotency evidence before scale-out. |
</threat_model>
