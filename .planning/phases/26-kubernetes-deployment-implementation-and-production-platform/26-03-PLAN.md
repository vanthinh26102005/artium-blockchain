<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet -> gateway Ingress | Public traffic must terminate at the gateway only and preserve websocket and webhook paths. |
| App pods -> private services/dependencies | Internal Services and stateful dependencies must not be publicly exposed. |
| Git repository -> Kubernetes Secrets | Only encrypted SealedSecret data or placeholders may be committed. |
| API deployments -> worker deployments | API pods must not run singleton processors; workers must run at explicit replica counts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-07 | Information Disclosure | SealedSecret examples | mitigate | Use `AgPLACEHOLDER_*` encrypted placeholders only and fail scripts when plaintext workflows are attempted. |
| T-26-08 | Elevation of Privilege | Ingress routing | mitigate | Render only gateway backend routing and forbid direct internal-service ingress. |
| T-26-09 | Denial of Service | probes/resources | mitigate | Require startup/readiness/liveness probes and resources on every Deployment. |
| T-26-10 | Tampering | NetworkPolicy | mitigate | Gate network policies through values and restrict service/dependency ingress paths. |
</threat_model>
