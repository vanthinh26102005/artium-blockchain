<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local validation -> real managed cluster | Static validation cannot prove DNS, TLS, ingress controller, storage class, or cloud load balancer behavior. |
| Sealed Secrets controller -> recoverability | Losing controller private keys can make committed SealedSecrets unrecoverable. |
| Stateful PVCs -> business continuity | In-cluster PostgreSQL, Redis, and RabbitMQ require backup/restore paths before production acceptance. |
| Rollout -> rollback | Failed Helm upgrades must have a documented rollback path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-11 | Repudiation | deployment validation | mitigate | Scripts log lint/render/schema/server-dry-run steps and docs define real-cluster acceptance. |
| T-26-12 | Denial of Service | stateful dependency loss | mitigate | Document PVC backup, logical backup, and restore drills for PostgreSQL, Redis, and RabbitMQ. |
| T-26-13 | Information Disclosure | Sealed Secrets key backup | mitigate | Document that controller key backups contain private keys and must be stored in the approved secure backup location. |
| T-26-14 | Tampering | rollback | mitigate | Provide `helm history` and explicit revision rollback helper. |
</threat_model>
