<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Helm values -> production cluster | Values drive live workloads, images, dependency placement, and exposure. Incorrect defaults can expose private services or deploy mutable images. |
| Legacy Kubernetes YAML -> production chart | Existing YAML is stale/dev-oriented and must not silently become production truth. |
| Chart dependencies -> stateful data | In-cluster PostgreSQL, Redis, and RabbitMQ store or broker critical runtime state and require persistence/resource/network defaults. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01 | Tampering | `Chart.yaml` and dependency versions | mitigate | Pin dependency versions and use Helm dependency build/lock rather than floating chart references. |
| T-26-02 | Information Disclosure | service exposure values | mitigate | Default every application Service to `ClusterIP`; public exposure is only the gateway Ingress in later plan templates. |
| T-26-03 | Denial of Service | in-cluster PostgreSQL/Redis/RabbitMQ | mitigate | Require persistence, explicit resources, probes, and network policy values in production overlay. |
| T-26-04 | Spoofing | images and tags | mitigate | Require immutable tag or digest placeholders and reject `latest`. |
</threat_model>
