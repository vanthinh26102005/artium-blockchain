<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| api-gateway auction DTO to browser | Public backend data becomes listing UI state |
| Socket.IO event to React state | Live events trigger data refreshes |
| local component state to listing card | Browser state could otherwise masquerade as auction truth |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-04 | T | `LiveAuctionPage.tsx` | mitigate | Do not update card current bids from local wallet events; call backend `refresh` after socket/polling signals. |
| T-17-05 | D | `useAuctionLots.ts` | mitigate | Normalize API errors and render retry UI instead of throwing during render. |
| T-17-06 | I | `auctionApis.ts` | accept | Auction listing data is public discovery content; avoid adding auth headers beyond standard `apiFetch` defaults unless backend requires them. |
</threat_model>
