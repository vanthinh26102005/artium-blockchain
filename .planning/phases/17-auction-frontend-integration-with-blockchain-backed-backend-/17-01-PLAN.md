<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| blockchain/order state to public auction DTO | Internal synchronized state is exposed to anonymous/public clients |
| RabbitMQ event to Socket.IO clients | Backend event messages become browser-visible live updates |
| artwork/order composition to frontend | Backend embeds artwork display metadata in auction DTOs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | I | `AuctionReadObject` | mitigate | Expose only auction presentation fields; do not include private shipping addresses, internal notes, collector IDs, or payment transaction IDs. |
| T-17-02 | T | `AuctionGateway` | mitigate | Broadcast only backend-refreshed snapshots or event-derived fields; do not accept client-provided bid amounts as server truth. |
| T-17-03 | D | `GetAuctionsHandler` | mitigate | Cap `take` to a safe maximum such as 50 and default `skip` to 0. |
</threat_model>
