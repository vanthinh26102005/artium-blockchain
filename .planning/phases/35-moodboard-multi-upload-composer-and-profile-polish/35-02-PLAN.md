<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Device files -> upload helper | Local file metadata is untrusted and must be validated before upload while backend remains authoritative. |
| Upload response -> create request | Backend-issued `mediaId` values are required before submit and are revalidated by backend create logic. |
| Preview object URLs -> browser memory | Every local preview URL must be revoked when no longer needed. |
| Queue state -> payload order | Visible order must match submitted `mediaIds` order. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-04 | Denial of Service | upload hook | mitigate | Enforce 1-10 files and mirror size/duration/MIME limits from shared upload API constants. |
| T-35-05 | Tampering | create payload | mitigate | Submit only uploaded `mediaIds` and a `coverMediaId` selected from uploaded queue items. |
| T-35-06 | Information Disclosure | preview URLs | mitigate | Revoke object URLs on remove/reset/unmount. |
| T-35-07 | Reliability | partial failure state | mitigate | Preserve successful uploads and metadata while failed files require retry or removal before create. |
</threat_model>
