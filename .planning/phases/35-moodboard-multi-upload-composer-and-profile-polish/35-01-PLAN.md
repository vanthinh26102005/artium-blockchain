<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Moodboard parent row -> child media rows | Query handlers attach child rows to parent records already selected by existing repository access rules. |
| Backend DTO -> frontend mapper | Frontend receives untrusted API data and must choose safe fallbacks instead of assuming every media row has every URL field. |
| Legacy cover URL -> uploaded media | Old moodboards may still depend on `coverImageUrl`, but new uploaded media must be preferred when present. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-01 | Information Disclosure | list/detail query handlers | mitigate | Only enrich moodboards returned by existing `findByUserId`, `findPublicByUserId`, or `findById`; do not broaden repository filters. |
| T-35-02 | Tampering | media order | mitigate | Trust persisted `displayOrder` from `MoodboardMediaRepository.findByMoodboardId` and sort deterministically in frontend mapping. |
| T-35-03 | Reliability | media URL mapping | mitigate | Mapper falls back from `thumbnailUrl` to `secureUrl` to `url` to legacy `coverImageUrl`/placeholder. |
</threat_model>
