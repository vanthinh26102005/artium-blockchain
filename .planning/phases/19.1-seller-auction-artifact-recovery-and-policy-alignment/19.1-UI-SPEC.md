# Phase 19.1 — UI Design Contract

> Visual and interaction contract for seller-auction artifact recovery and policy alignment. This phase preserves the existing Phase 19 seller workspace, adds only the pre-start policy-summary refinements needed to close SAUC-04/SAUC-05 wording gaps, and does not begin Phase 20 auction start orchestration.

**Sources used:** `REQUIREMENTS.md`, `ROADMAP.md`, `v1.1-MILESTONE-AUDIT.md`, existing Phase 18/19 UI-SPECs, `components.json`, `tailwind.config.js`, current seller auction workspace code, current seller registration route.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn-compatible local primitives in `@shared/components/ui` |
| Preset | `radix-lyra` with existing Artium seller workspace styling |
| Component library | Radix UI primitives via local shared components |
| Icon library | `lucide-react` for seller-auction surfaces |
| Font | `ABC Monument Grotesk` for page chrome; preserve existing `Inter` only inside reused legacy components |

### Required Existing Patterns

- Keep the existing Phase 19 route: `/artist/auctions/create`.
- Keep the existing two-step seller workspace and selected-artwork summary.
- Preserve the current white/off-white seller auction workspace, blue accent, rounded panels, and pill-button language.
- Keep the selected artwork summary plus the preview/policy area as the main focal point of the terms step; the page should visually direct attention there before the primary CTA.
- Preserve the current `/seller/register` page visually; Phase 19.1 restores evidence for SAUC-10 but does not redesign seller registration.
- Do not add pending/active/failed/retryable start-status UI, tx hash UI, or wallet orchestration UI. Those remain Phase 20 only.
- Do not add a new route, modal, or wizard step for activation.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Seller registration route | Preserve current `/seller/register` flow. No new CTA logic, modal, or redirect pattern in this phase. |
| Terms step entry | Preserve current Phase 19 behavior: artwork selection first, then terms + preview. |
| Fee disclosure visibility | The fee/policy block must be visible in the preview/policy area whenever the terms step is visible. Do not hide it behind an accordion, tooltip, or secondary click. |
| Missing dynamic fee data | If no numeric fee source exists, render the static fee policy copy above. Do not omit the fee section. |
| Start Auction click | Validate locally, keep the seller on the same page, and do not call wallet, backend, or on-chain start flows. |
| Save Draft | Preserve current local-only behavior. Do not imply backend persistence. |
| Preview tone | Use `preview`, `before activation`, and `next phase` language. Never imply the auction is already live, started, or confirmed. |
| Phase 20 boundary | Do not add tx hash, pending, active, failed, retry, or lifecycle status surfaces. |
