## Summary

Phase 19.1 is primarily a **gap-closure and evidence-repair phase**, not a new feature phase. The repository already contains a working seller registration route and a working Phase 19 seller auction workspace, but the artifact trail is incomplete: Phase 18.1 has only `PLAN.md`, and Phase 19 still has no `19-VERIFICATION.md`.

The **minimal UI delta** is small. The current seller workspace already satisfies most of SAUC-04/05 behavior, and the approved 19.1 UI-SPEC narrows the remaining work to:

1. Add an explicit fee/policy disclosure block in the preview.
2. Update one policy card to include fees plus lock-after-activation wording.
3. Align helper copy from “next step” to “next phase” so Phase 20 boundaries stay explicit.

Do **not** start Phase 20 here. Current `Start Auction` behavior is intentionally local-only validation, and both the approved UI-SPEC and current codebase preserve that boundary.

## Existing Patterns to Reuse

### Pattern 1: Reuse the existing server-authoritative seller activation flow

Keep seller role activation in the backend create-seller-profile transaction; treat 19.1 as evidence recovery, not behavior redesign.

### Pattern 2: Preserve the local-validation start boundary

`Start Auction` should validate locally and stop; no wallet, backend, or contract call belongs in this phase.

### Pattern 3: Close SAUC-05 with static policy copy

Add a visible fee/policy disclosure block inside the preview/policy area using the approved 19.1 copy. Do not invent numeric fees or a fee-calculation engine.

## Pitfalls

1. **Do not turn 19.1 into Phase 20.** No tx hash UI, no pending/active/failed status, no wallet orchestration.
2. **Do not re-implement seller registration.** Existing code path already exists.
3. **Do not compute or invent a numeric seller fee.** The approved contract only requires static fee/gas wording.
4. **Do not use repo-wide lint as the phase gate.** Prior phases already documented unrelated frontend lint noise.
5. **Do not over-claim SAUC-10 atomicity.** The backend seller-profile creation + seller-role activation is transactional; FE display-name/slug sync is separate.
