<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Device file -> frontend upload helper | The browser receives untrusted local file metadata and must validate before upload while relying on backend validation as authoritative. |
| Upload response -> create request | `mediaId` is a user-controlled value after upload and must remain backend-verified during moment creation. |
| Object URL preview -> browser memory | Preview URLs must be revoked to avoid leaks during replace/close/unmount. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-01 | Spoofing | create payload | mitigate | Composer emits `CreateMomentInput` with `mediaId`; backend verifies owner/status/context from Phase 33. |
| T-34-02 | Denial of Service | upload flow | mitigate | Client mirrors MIME/size/duration checks from `profileMediaUploadApi.ts` before upload. |
| T-34-03 | Information Disclosure | preview object URL | mitigate | Hook revokes preview URLs on replace, reset, and unmount. |
| T-34-04 | Tampering | stale replacement upload | mitigate | Hook clears old `uploadedMedia` and `mediaId` before replacement upload starts. |
</threat_model>
