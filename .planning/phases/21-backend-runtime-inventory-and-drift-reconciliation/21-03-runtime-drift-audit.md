# 21-03 Runtime Drift Audit

## Scope

This audit reconciles the Phase 21 runtime inventory against direct repository evidence from service bootstraps, Dockerfiles, Compose, and legacy Kubernetes manifests. It records current drift only. It does not choose target production topology, ingress, service exposure, scaling, or managed dependency policy.

## Confirmed mismatches

- `api-gateway`: active runtime path is `8081`; Dockerfile still says `EXPOSE 3002`.
- `artwork-service`: Compose publishes `3002`, inventory records env `PORT=3002`, but code falls back to `3003` and legacy K8s uses `containerPort: 3003`.
- `payments-service`: Compose publishes `3003`, inventory records env `PORT=3003`, but code, Dockerfile, and legacy K8s use `3005`.
- `messaging-service`: Compose publishes `3005`, inventory records env `PORT=3005`, but code, Dockerfile, and legacy K8s use `3004`.
- `orders-service`: code and Compose use `3004`; Dockerfile says `EXPOSE 3006`.
- `notifications-service`: Compose publishes `3006`, inventory records `.env.compose` `PORT=3006`, code uses `process.env.port ?? 3002`, Dockerfile says `EXPOSE 3003`, and legacy K8s service/deployment ports do not agree.
- `artwork` K8s uses `artworks-service` and `dev:artworks` while the repository service is `artwork-service` and Compose uses `dev:artwork`.
- Current checkout contains no readable `.env.compose` files even though Compose references them and earlier inventory artifacts cite their values.

## Legacy or dev-only artifact findings

- Legacy Kubernetes manifests use live-development commands (`yarn run dev:*`) instead of immutable-image startup commands.
- Multiple Kubernetes deployments mount `/app` and node modules through `emptyDir`, which is development evidence rather than a production runtime contract.
- Several Kubernetes Services use `NodePort` before this milestone has chosen public exposure rules.
- RabbitMQ, Mailhog, and service-level `NodePort` resources should be treated as historical/dev exposure examples until Phase 23 defines the exposure model.
- The legacy artwork manifests use pluralized `artworks-service` naming that does not match the current repo.
- Volume mount names such as `node-modules` do not match declared volumes such as `identity-node-modules`, `payments-node-modules`, `artwork-node-modules`, and `messaging-node-modules`.
