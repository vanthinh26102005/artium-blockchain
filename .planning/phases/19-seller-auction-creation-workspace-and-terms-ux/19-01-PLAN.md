<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser form to local validation | Seller-controlled strings are validated before preview/start handoff state changes. |
| browser localStorage | Draft data is stored on the user's device and can be stale or tampered with. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Tampering | `validateSellerAuctionTerms` | mitigate | Treat local validation as UX only; Phase 20 backend must revalidate all economics before activation. |
| T-19-02 | Information Disclosure | local draft storage | mitigate | Store only selected artwork terms/disclosures; no wallet secrets, auth tokens, order IDs, or private buyer data. |
| T-19-03 | Repudiation | local draft wording | mitigate | UI copy must say `Draft saved on this device.` and never claim backend persistence. |
</threat_model>
