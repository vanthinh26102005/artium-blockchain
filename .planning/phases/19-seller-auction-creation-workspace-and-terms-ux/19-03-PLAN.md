## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| backend candidates to two-step UI | Server-owned eligibility remains the only source for eligible/blocked artwork selection. |
| validated local terms to future activation | Phase 19 validated values are local preview input, not authoritative auction state. |
| seller click to start/handoff | User intent to start must not trigger unsupported backend/on-chain side effects in Phase 19. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-07 | Spoofing | frontend seller gate | accept with mitigation | Keep Phase 18 backend seller endpoint authoritative; frontend role gate is guidance only. |
| T-19-08 | Tampering | selected artwork and terms state | mitigate | Derive selectable artwork from `eligible` candidates only and never accept client-supplied seller identity. |
| T-19-09 | Elevation of Privilege | start action | mitigate | Do not call wallet, contract, or backend start APIs in Phase 19; Phase 20 owns server authorization. |
| T-19-10 | Information Disclosure | local draft and preview | mitigate | Store only seller-entered auction disclosures and numeric terms; no auth tokens, buyer data, or wallet secrets. |
</threat_model>
