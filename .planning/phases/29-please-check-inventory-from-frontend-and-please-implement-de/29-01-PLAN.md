<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> API gateway | The browser can request update/delete/visibility changes for any artwork ID. |
| API gateway -> artwork-service RPC | Other producers can send RPC messages, so artwork-service must enforce ownership. |
| Artwork-service -> lifecycle projection | Local lifecycle snapshots are asynchronous projections and must be treated as lock signals when present. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-01 | Information Disclosure | profile list | mitigate | Add `isPublished` query filtering and require profile pages to query `status: ACTIVE` plus `isPublished: true`. |
| T-29-02 | Elevation of Privilege | update/delete handlers | mitigate | Require `command.user.id` and compare to `existingArtwork.sellerId` in artwork-service handlers. |
| T-29-03 | Tampering | auction-locked artwork | mitigate | Block update/delete when seller auction lifecycle indicates pending, active, retry, or non-editable failure. |
</threat_model>
