<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Inventory UI -> artwork API | User-triggered menu actions mutate server state and must use authenticated APIs. |
| Inventory UI -> auction workspace | Inventory can route to auction setup but must not directly start wallet/RPC flows. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-04 | Tampering | profile visibility toggle | mitigate | Send explicit `{ status, isPublished }` patch to backend and update UI from returned API object only. |
| T-29-05 | Repudiation | delete modal | mitigate | Require modal confirmation and disable repeated confirm while request is in flight. |
| T-29-06 | Elevation of Privilege | auction start from inventory | mitigate | Inventory only navigates to seller auction workspace and never invokes start-attempt or wallet APIs. |
</threat_model>
