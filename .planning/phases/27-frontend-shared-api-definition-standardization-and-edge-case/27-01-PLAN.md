<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser runtime -> API gateway | The shared client decides auth headers, request body format, and URL composition for every frontend API call. |
| Backend error payload -> UI/domain code | Error parsing determines what downstream code can inspect and display. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-01 | Tampering | query/path construction | mitigate | Centralize query encoding and path segment encoding so user-controlled values cannot alter route structure. |
| T-27-02 | Information Disclosure | structured API errors | mitigate | Preserve backend error data for programmatic handling without logging secrets in the shared client. |
| T-27-03 | Denial of Service | response parsing | mitigate | Handle empty responses and non-JSON bodies without throwing parse errors that trigger retry loops. |
</threat_model>
