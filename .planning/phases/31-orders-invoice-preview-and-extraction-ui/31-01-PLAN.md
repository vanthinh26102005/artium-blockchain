<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backend invoice DTO -> browser UI | Invoice components receive financial truth from `OrderInvoiceResponse`; they must not recompute totals or invent missing fields. |
| Authorized/redacted data -> shared components | Seller-redacted values must remain labeled absences, not inferred from local order data. |
| Modal/print controls -> browser print | Print should expose only invoice document content, not private workspace chrome. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-01 | Tampering | `OrderInvoiceDocument` | mitigate | Render subtotal, tax, discount, shipping, total, invoice number, and status only from `OrderInvoiceResponse` fields. |
| T-31-02 | Information Disclosure | availability helper | mitigate | Map authorization/not-found/forbidden errors to `Invoice unavailable for this workspace.` without exposing backend distinctions. |
| T-31-03 | Information Disclosure | print/modal | mitigate | Keep print document scoped to `.order-invoice-print-root`; Wave 2 adds print CSS to hide app chrome and controls. |
</threat_model>
