<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Orders list/detail -> invoice endpoint | UI can initiate invoice reads for arbitrary order IDs present in the authenticated workspace; backend remains the authorization source. |
| Backend error -> user-facing state | Raw authorization/not-found errors must become non-disclosing unavailable UI. |
| App workspace -> print output | Print must not expose surrounding workspace UI, filters, pagination, or modal frame. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-04 | Information Disclosure | `OrderListCard` and `OrderDetailPageView` | mitigate | Preserve backend authorization by calling only `orderApis.getOrderInvoice(order.id)` and render authorization errors as `Invoice unavailable for this workspace.` |
| T-31-05 | Tampering | detail invoice state | mitigate | Do not derive invoice totals/status/identity from `order`; pass `invoice` DTO to presentation components. |
| T-31-06 | Information Disclosure | browser print | mitigate | Add scoped print CSS so only `.order-invoice-print-root` and minimal footer print, while controls and workspace chrome are hidden. |
</threat_model>
