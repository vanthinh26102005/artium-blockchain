<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated seller request -> orders-service orchestration | Seller-owned auction starts must stay seller-scoped and idempotent across retries and refreshes. |
| backend lifecycle state -> FE wallet handoff | The backend must shape the transaction request without becoming the signer. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-01 | Elevation of Privilege | seller start routes | mitigate | Guard every route with seller auth/role checks and revalidate ownership/eligibility in the command handler. |
| T-20-02 | Tampering | canonical attempt / order ID reuse | mitigate | Enforce one open attempt per seller+artwork and bind retries to the same canonical order ID. |
| T-20-03 | Spoofing | wallet attachment | mitigate | Require stored wallet readiness and reject attach-tx calls whose connected wallet does not match the persisted seller wallet. |
| T-20-04 | Information Disclosure | seller lifecycle status | mitigate | Keep status queries seller-scoped and do not surface pending/failed attempts in public auction endpoints. |
</threat_model>
