## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FE form state -> backend start lifecycle | Local validation must not be confused with persisted lifecycle state. |
| FE wallet provider -> seller lifecycle UI | Wallet/network failures must be surfaced explicitly without enabling duplicate or unsafe actions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-05 | Tampering | locked seller controls | mitigate | Disable unsafe economics and artwork edits immediately after valid submit and keep them locked while pending or active. |
| T-20-06 | Denial of Service | duplicate submit path | mitigate | Persist lifecycle on the backend and rehydrate it on refresh so the FE cannot re-enable `Start Auction` during pending work. |
| T-20-07 | Information Disclosure | lifecycle shell actions | mitigate | Show tx hash, reason code, and next-step guidance only on seller-owned lifecycle surfaces with visible labels/accessibility names. |
| T-20-08 | Spoofing | wallet handoff | mitigate | Require MetaMask/Sepolia checks and surface explicit wallet mismatch/network errors instead of silently retrying or downgrading behavior. |
</threat_model>
