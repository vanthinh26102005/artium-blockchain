<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser to gateway | User-controlled HTTP request reaches seller candidate endpoint. |
| gateway to microservices | Gateway sends trusted sellerId to artwork/orders services. |
| orders to eligibility response | Active order existence becomes reason code without exposing private order data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Spoofing | `GET /auctions/seller/artwork-candidates` | mitigate | Require `JwtAuthGuard`, `RolesGuard`, and `@Roles(UserRole.SELLER)`. |
| T-18-03 | Tampering | seller identity selection | mitigate | Derive seller from `req.user.id`; never accept client `sellerId`. |
| T-18-04 | Information Disclosure | `ACTIVE_ORDER_LOCK` merge | mitigate | Return only the lock reason, not order IDs, buyer IDs, payment IDs, or shipping details. |
| T-18-06 | Denial of Service | order lock lookup | mitigate | Query only artwork IDs returned by seller-owned candidate lookup; return empty locks when candidate list is empty. |
</threat_model>
