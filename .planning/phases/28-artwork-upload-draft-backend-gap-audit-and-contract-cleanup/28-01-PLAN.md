<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser route -> gateway draft routes | The `draftArtworkId` query value is user-controlled and must be owner-scoped before loading or mutating draft data. |
| Gateway request body -> artwork-service | Client payloads must not be trusted for seller identity. |
| Draft state -> published artwork | Invalid or incomplete draft data must not transition into public inventory. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-01 | Spoofing | draft routes | mitigate | Derive seller identity from `UserPayload.id` and reject body-driven `sellerId`. |
| T-28-02 | Information Disclosure | draft read | mitigate | Return draft only when `artwork.sellerId === user.id` and `status === DRAFT`. |
| T-28-03 | Tampering | draft submit | mitigate | Validate owner, draft status, required fields, and primary image before lifecycle transition. |
</threat_model>
