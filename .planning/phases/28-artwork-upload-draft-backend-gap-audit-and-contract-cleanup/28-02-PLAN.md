<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Authenticated browser -> upload gateway | Multipart form data is user-controlled and must not define seller identity. |
| Uploaded file -> GCS path | Files must be keyed only after draft ownership is verified. |
| Backend draft -> frontend form | Draft load failures must not be hidden by local fallback state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-04 | Spoofing | upload seller ID | mitigate | Derive `sellerId` from `req.user.id` in gateway upload handlers. |
| T-28-05 | Tampering | upload artwork ID | mitigate | Artwork-service checks draft existence, owner, and `DRAFT` status before upload. |
| T-28-06 | Information Disclosure | draft hydration | mitigate | Frontend surfaces 403/404 draft load states instead of rendering stale local draft data. |
</threat_model>
