<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser file selection -> frontend helper | Client validation improves UX but cannot be trusted for security. |
| Frontend helper -> gateway upload | Multipart requests must carry auth and avoid manual `Content-Type`. |
| Frontend create body -> backend | `mediaId` values are still user-controlled and backend must verify ownership/status. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-08 | Tampering | frontend validation | mitigate | Mirror backend MIME, size, duration, and count constants, while treating backend as authoritative. |
| T-33-09 | Information Disclosure | upload transport | mitigate | Reuse `apiUpload` auth handling and structured errors; do not bypass shared client behavior. |
| T-33-10 | Spoofing | create request types | mitigate | Remove URL-proof request fields and expose `mediaId`/`mediaIds` only. |
</threat_model>
