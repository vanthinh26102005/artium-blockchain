<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> gateway multipart upload | File content, filename, MIME type, size, and duration metadata are user-controlled. |
| Gateway -> community-service RPC | Gateway derives `userId` from auth and passes file buffers to the service that owns media records. |
| Community-service -> GCS | Storage paths and object metadata must not leak into artwork semantics or cross-user folders. |
| Community-service -> database | Pending media records are the proof consumed by later create commands. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-01 | Spoofing | upload routes | mitigate | Derive owner from `req.user.id`; never accept body `userId`. |
| T-33-02 | Tampering | file validation | mitigate | Enforce exact MIME, count, and size constants in community-service before upload. |
| T-33-03 | Information Disclosure | storage paths | mitigate | Store under `community/{userId}/...` paths and do not expose other users' media rows. |
| T-33-04 | Denial of Service | moodboard batch upload | mitigate | Cap moodboard upload batch at 10 files and video files at 100MB. |
</threat_model>
