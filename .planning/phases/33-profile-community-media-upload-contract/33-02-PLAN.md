<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser create body -> gateway | `mediaId`, `mediaIds`, and `coverMediaId` are user-controlled IDs and must be verified server-side. |
| Gateway -> community-service | `userId` is derived from auth and must remain the creator authority. |
| Pending media -> content rows | Pending uploaded media becomes public profile content only after owner/status/context checks. |
| Moodboard media -> moodboard gallery | Uploaded media items must be ordered and cover-selected without overloading artwork references. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-05 | Elevation of Privilege | creation commands | mitigate | Verify uploaded media owner equals authenticated `userId` before content creation. |
| T-33-06 | Tampering | moodboard media ordering | mitigate | Persist media rows server-side with deterministic `displayOrder` and selected cover. |
| T-33-07 | Information Disclosure | create APIs | mitigate | Reject media IDs that do not belong to the current user instead of resolving arbitrary URLs. |
</threat_model>
