# v1.1 Seller Registration Gap Audit

**Date:** 2026-04-25
**Scope:** Profile settings access path for seller auction creation
**Trigger:** `$gsd-plan-milestone-gaps` request to review `/profile/` files and implement register-as-seller validation.
**Status:** gap_closed

## Original Business Intent

Phase 18 established seller-only auction artwork selection. Practical access requires a non-seller account to have a safe path from profile settings to become seller-capable before visiting seller-only auction pages.

## Gap Findings

| Gap | Severity | Evidence | Required Closure |
|-----|----------|----------|------------------|
| Missing seller registration route | High | `ProfileEditPage` CTA navigated to `/seller/register`, but no `FE/artium-web/src/pages/seller/register.tsx` existed. | Add protected seller registration route and page. |
| Seller role activation not guaranteed by registration | High | `CreateSellerProfileHandler` created `seller_profiles` but did not update `users.roles`. Seller-only auction access depends on `seller` capability. | Add `seller` role server-side during seller profile creation. |
| Profile registration policy not explicit | Medium | Profile CTA copy implied unlocking seller features, but did not clarify verification/payment onboarding policy. | Add UI copy and defaults showing registration is active but unverified and payment onboarding incomplete. |
| Duplicate registration recovery missing | Medium | Existing seller profile lookup existed, but no UX route handled already-registered sellers from the CTA. | Detect existing seller profile and route user back to profile setup. |

## Closure Phase

Created Phase 18.1: Seller registration and role activation gap closure.

## Acceptance Evidence Required

- `/seller/register` exists and is auth-protected.
- Seller registration form validates display name, profile URL, seller type, bio, location, and website URL.
- Backend creates seller profile for authenticated `userId`, not client-provided `userId`.
- Backend adds `UserRole.SELLER` in the same database transaction as seller profile creation.
- New seller profile defaults remain `isActive: true`, `isVerified: false`, payment onboarding false.
- Duplicate seller profile creation returns conflict and frontend can recover to existing setup.
