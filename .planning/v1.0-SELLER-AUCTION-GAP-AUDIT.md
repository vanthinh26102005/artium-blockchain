# Seller Auction Creation Gap Audit

## Finding

The buyer-facing auction experience now has public listing, realtime refresh, and wallet-backed bidding. The seller-facing creation side is still missing: sellers cannot pick one of their artworks, configure auction terms, and start an auction through a protected workflow.

## Practical Policy Recommendations

- Authorization must be enforced by backend guards; frontend route protection is only a UX layer.
- Only the artwork owner can start an auction.
- Eligible artworks should be single-quantity, active/complete inventory records with primary media and no active order, sale, or auction lock.
- Auction terms should be validated before chain interaction: start bid, reserve policy, minimum increment, duration, start/end timing, and shipping/payment disclosures.
- After activation, seller-editable economics should be locked. Any later lifecycle action must be explicit and state-valid.
- On-chain transaction hash is pending evidence only. Public listing and seller status should rely on backend/on-chain synchronized state.

## Routing

Created milestone v1.1 requirements and roadmap phases 18-20 to close this gap.
